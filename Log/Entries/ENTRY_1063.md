FOSSIL_n → REPAIR_n+1 (context delta) → VERIFY → LOCK

2️⃣ Recurse, Don’t Rewrite

Flux is handled through recursive audit: the new entry tests itself against the old truth. If it passes, both coexist; if not, the new one remains pending under [REPAIR].
